1. Scope of var, let, and const:

In JavaScript, variables declared with var have function scope, which means they are accessible throughout the entire function where they are defined. On the other hand, variables declared with let and const have block scope, limiting their accessibility to the block of code where they are declared.

Var:

    
function varScope() {
  if (true) {
    var x = 10;
  }

  console.log(x); // 10
}
varScope();
console.log(x); // undefined

let:

    
function letUsage() {
  let A = "Hello, ";
  if (true) {
  let A = "World!";  
  console.log("Inside:", A); 
  // World!   
  }
  console.log("Outside:", A); 
  // Hello, 
}
letUsage();

const:

    
function ConstUsage() {
  const pi = 3.1416;
  console.log(pi);
  // Attempting to reassign const
  //It will result in an error.
  // pi = 3.14; TypeError
  // we can not assign new value
}
ConstUsage();

2. Use cases of null and undefined:

null is used to intentionally represent the absence of a value or to clear a variable or property.

undefined typically means a variable exists but has not been given a value, or a function doesn't return anything.

Undefined vs Null

Area Undefined Null
Definition Undefined means the variable has been declared, but its value has not been assigned. Null means an empty value or a blank value.
The typeof operator The typeof() operator returns undefined for an undefined variable. The typeof() operator returns the type as an object for a variable whose value is assigned as null.
On performing arithmetic operations It returns NaN on performing arithmetic operations. Converts to 0 then performs the operation.
Is it an assignment value? No, as there is no value assigned to the variable, it becomes undefined. Yes, as we assign null to a variable, it is an assignment value.

3. What is a REST API?

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. REST is not a specific technology or protocol, but rather a set of constraints and principles for building web services that are scalable, stateless, and can be easily consumed by clients

Key Principles of REST API

  1. Stateless: Each request from a client to a server must contain all the information needed to understand and process the request.
  2. Client-Server Architecture: REST separates the client and server, allowing them to evolve independently.
  3. Uniform Interface: It uses standard HTTP methods for interaction.
  4. Resource-Based: Resources are identified by URLs and manipulated using HTTP methods.
  5. Representation: Resources can have multiple representations like JSON or XML.
  6. Statelessness: The server does not store client state between requests.
  7. Layered System: It can be composed of multiple layers to improve performance and security.
